Solid Company Settings - Version 1.2

Description

Admin users can easily update their company information and have those changes instantly reflected on the website.

Available Fields

  • Company Name
  • Company Type (defined by Schema.org)
  • Slogan
  • Logo
  • Domain Name
  • Phone
  • Phone 2
  • Phone 3
  • Fax
  • Email
  • Address (2 locations)

New In This Version

  • Added additional phone numbers for each location
  • Added emial to schema definition
  • Bug Fix: Updated schema output to read "type" field
  • Bug Fix: Updated schema output to read "domain" field

Requirements

  • Developer to use the proper tags throughout the website
  • Liquid markup enabled on website

Developer Notes

Tag to Enable App on Website

Place in the head of the template

Standard: {%include "/_System/apps/solid-company-settings/_data/company.inc" -%}
Rich-Snippets: {%include "/_System/apps/solid-company-settings/_data/company-rich.inc" -%}
Rich-Snippets (JSON): {%include "/_System/apps/solid-company-settings/_data/company-rich-json.inc" -%}

Both tags will make all the company data tags avaiable, the difference is in the output:

  • Standard tag: No output
  • Rich Snippet tag: Will output all the company information in Schema.org Microdata markup using meta tags (no information will be displayed to the user)
  • Rich Snippet JSON tag: Will output all the company information in Schema.org Microdata markup using JSON-ld markup (no information will be displayed to the user)

Company Data Tags can be used within

  • Templates
  • Pages
  • Files referenced through Includes (see below)
  • Content Holders

Use Company Data Tags inside Content Holders

The best way to use these tags inside content holders is to replace your content holders with TPL or include files and then reference these files using the Liquid Include tags.

Example:

USE: {%include "/_System/Includes/_Template - Header.inc"%}
NOT: {module_contentholder, name="_Template - Header"}

If you want to use the company tags inside content holders you will need to place the main include file in the content holder as liquid variables do not get passed in and our of content holders
